# News Trends Analytics Pipeline

## Checkpoint Presentation Content

### Slide 1 — Project Overview

**Title:** News Trends Analytics Pipeline

**Problem Statement:**
News is published continuously across many sources, making it difficult to track patterns, major topics, and source activity in a structured way. This project aims to build an automated data engineering pipeline that collects live news headlines, stores them in a structured database, cleans and validates the data, and serves analytical insights through a dashboard.

**Data Source:**

* NewsAPI (real-time news headlines and article metadata)

**Expected Outcomes:**

* Automated ingestion of live news data
* Structured storage of article metadata
* Data cleaning and validation
* Analytical insights such as source frequency, category trends, and keyword patterns
* Dashboard-based presentation of results

---

### Slide 2 — Architecture Design

**Pipeline Architecture:**

NewsAPI → Ingestion Script → SQLite Database → Data Cleaning & Validation → Analytics Layer → Streamlit Dashboard
                                                                                         ↓
                                                                                     Prefect Orchestration

**Technology Stack:**

* Python for ingestion and processing
* Requests for API calls
* SQLite for structured storage
* Pandas for transformation and analysis
* Prefect for orchestration and scheduling
* Streamlit for serving visual insights

**Design Rationale:**

* Modular structure makes the project easier to maintain
* SQLite is lightweight and easy to integrate
* Prefect supports retries, scheduling, and observable flows
* Streamlit enables fast development of an interactive dashboard

* url TEXT UNIQUE
